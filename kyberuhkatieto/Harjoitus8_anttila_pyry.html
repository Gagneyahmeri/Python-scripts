Subplots

Kuvaajien vertailtavuuden helpottamiseksi kuvaajia kannattaa opetella ryhmittelemn vierekkin ja allekain. Tss tehtvss opetellaan tekemn piirakkakaavio ja asettamaan kaksi kuvaajaa vierekkin.

Lue datasetti 'cybersecyrity_attacks.csv' DataFrameen urlista https://student.labranet.jamk.fi/~ajota/Datasets/cybersecurity_attacks.csv.

Tee seuraavat kaaviot samaan kuvaan vierekkin, mallin net alla.

  1. Visualisoi piirakkakaavioon eri hykkystyyppien prosentuaaliset osuudet datasetiss.
  2. Visualisoi pylvskaavioon hykkystyypin 'Malware' lukumrt kuukausittain vuonna 2022. (Vinkki: Tee ensin "Timestamp"-sarakkeesta uudet sarakkeet "Year" ja "Month" ja lhde suodattamaan ja ryhmittelemn dataa niden avulla.

# Vastaus
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv('https://student.labranet.jamk.fi/~ajota/Datasets/cybersecurity_attacks.csv', index_col=0)

# Datan ksittely
df['Timestamp'] = pd.to_datetime(df['Timestamp'])

df['Year'] = df['Timestamp'].dt.year
df['Month'] = df['Timestamp'].dt.month

atcount = df['Attack Type'].value_counts()
atpercentage = atcount / atcount.sum() * 100

malware = df[(df['Year'] == 2022) & (df['Attack Type'] == 'Malware')].groupby('Month').size()

# Kuvaajat
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

axes[0].pie(atpercentage, labels=atpercentage.index, autopct='%1.1f%%')
axes[0].set_title('Hykkystyyppien prosentuaaliset osuudet datassa')

axes[1].bar(malware.index, malware.values)
axes[1].set_title('Hykkystyypin "Malware" lukumrt kuukausittain vuonna 2022')

axes[1].set_xticks(range(1, 13))

plt.show()

import matplotlib.dates as mdates

df_rpa = pd.read_csv('https://student.labranet.jamk.fi/~ajota/Datasets/rba-dataset_january2021.csv', index_col=0)

# Datan ksittely
df_rpa['Login Timestamp'] = pd.to_datetime(df_rpa['Login Timestamp'])

df_rpa.set_index('Login Timestamp', inplace=True)

successful_logins = df_rpa.resample('h')['Login Successful'].sum()
unsuccessful_logins = df_rpa.resample('h')['Login Successful'].apply(lambda x: (x == False).sum())

login_counts = pd.DataFrame({'True': successful_logins,'False': unsuccessful_logins}).fillna(0)

# Kuvaaja
plt.figure(figsize=(12, 6))
plt.plot(login_counts.index, login_counts['True'], label='True')
plt.plot(login_counts.index, login_counts['False'], label='False')

ax = plt.gca()
ax.xaxis.set_major_locator(mdates.HourLocator(interval=20))
ax.xaxis.set_major_formatter(mdates.DateFormatter('%m-%d %H'))

plt.title('Onnistuneet ja eponnistuneet kirjautumiset')
plt.xlabel('Pivmr ja Kellonaika')
plt.ylabel('Kirjautumisten Mr')
plt.legend()
plt.xticks(rotation=90)
plt.margins(x=0)
plt.show()

Keskittyvtk kirjausyritykset tietyille piville tai tunneille? Mik selitt tt?

  • Kirjausyritykset nyttvt noudattavan aika snnllist rytmi. Mitn tietty piv kuvaajan perusteella ei voi sanoa, mutta tunneista voi sanoa, ett pivsaikaan tulee enemmn.

Onko datassa havaittavissa trendej tai kausivaihtelua?

  • Datassa nkyy selkesti ett alkuvuodesta on ollut huomattavasti enemmn eponnistuneita kirjautumisyrityksi, kuin loppuvuodesta. Onnistuneet kirjautumiset ovat pysyneet suurinpiirtein samalla tasolla.

Miten onnistuneiden ja eponnistuneiden kirjautumisyritysten piikit vertautuvat toisiinsa? Lytyyk tekemllesi havainnolle jokin luonnollinen selitys?

  • Onnistuneiden ja eponnistuiden kirjautumismrt nousevat ja laskevat snnlliseti ja se johtuu todennkisesti, ett kyttjmrt ovat siihen aikaan pienemmt, sill ihmiset ovat nukkumassa.
